Everything a real blog needs on day one

Extracted from a production blog running 120+ posts, then hardened through five external review rounds.

📄File-based MDX

Posts are .mdx files with a typed frontmatter contract — parsed with gray-matter, rendered server-side with next-mdx-remote. No CMS, no database.

🗂Curated topic hubs

Tags map into a handful of canonical categories instead of one thin page per raw tag. Empty hubs 404 automatically; a build-time gate catches orphaned posts.

🔍Serious SEO layer

Article, FAQPage, CollectionPage, and BreadcrumbList JSON-LD; canonical URLs; OG/Twitter metadata; RSS; a one-line sitemap helper; self-canonical paginated archives.

📚Article UX built in

Auto table of contents with GitHub-style deduped anchors, breadcrumbs, author bio, share buttons, and tag-scored related posts.

🧩MDX components

<Callout>, <InfoCard>, <QuickRef>, <Verdict>, <Divider> — available in every post without imports.

📣Conversion CTAs

Inline, bottom, and sticky-sidebar CTA blocks with named copy variants selectable per post, an analytics callback hook, and a single off switch — all in one config file.

🩺Self-diagnosing

npx next-blog-kit doctor checks deps, wiring, frontmatter, duplicate/reserved slugs, missing hero images, broken internal links, orphaned posts, and empty hubs.

🛡Safe installer

All-or-nothing writes: any file collision aborts the whole install with a report. --force backs up every overwritten file first.

✍️AI authoring skill

An optional Claude Code skill runs a full editorial pipeline — topic gate, SEO/AEO-structured draft, taxonomy check, and AI-generated hero images.

Five-minute install

Works with fresh create-next-app projects and existing apps, root or src/ layouts.

npx next-blog-kit init — routes, components, lib, content scaffold, and styles land in your repo.

One config drives everything

// lib/blog/config.ts
export const blogConfig = {
  siteName: "My Site",
  siteUrl: "https://example.com",
  categories: [
    { slug: "guides", title: "Guides",
      tags: ["how-to", "tutorial"] },
  ],
  cta: {
    enabled: true,
    href: "/signup",
    variants: { default: { /* copy */ } },
  },
  // optional analytics hook
  onCtaClick: (e) => posthog.capture("blog_cta", e),
};

Articles that write themselves (with editorial standards)

npx next-blog-kit skill installs a Claude Code skill distilled from a real automated content pipeline — not a "generate text" wrapper.

1 · Topic gate

Searches your existing corpus first and classifies the work as net-new, refresh, or consolidate — it stops rather than create a duplicate-intent URL.

2 · Draft with rules

1,500–2,500 word MDX following the frontmatter contract, SEO/AEO structure, internal links, FAQ + JSON-LD sync, and hard anti-AI-slop writing rules.

3 · Taxonomy check

Verifies tags land the post in at least one topic hub before it ships — the same gate the doctor and the runtime enforce.

4 · Hero image

Provider ladder: any image-gen MCP tool → Leonardo.ai → OpenAI → a detailed manual prompt. Always a 16:9 JPEG under 400 KB with real alt text.

Requirements & trust model

Honest constraints, stated up front.

  • Next.js 15+ (App Router) — routes use Promise-based params. Continuously tested against 15 and latest in CI, including a timezone-shifted production build.
  • Tailwind CSS 4 + @tailwindcss/typography. Components use shadcn-style semantic tokens; a documented fallback block covers apps without them.
  • A @/* path alias (the create-next-app default). The installer checks and tells you if it's missing.
  • MDX is code. Posts can run arbitrary JSX at build time — by design, for content you author. Don't pipe untrusted user submissions into content/blog/.
  • You own the code. This is a copy-in, not a dependency: upgrade by choice, customize without forking, remove by deleting the files.
